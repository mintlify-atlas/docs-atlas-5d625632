---
title: "Quickstart"
description: "Get the First Lego League Backend running locally in minutes."
---

<Steps>
  <Step title="Prerequisites">
    Ensure you have the following installed:

    - **Java 21** (IBM Semeru or any JDK 21-compatible distribution)
    - **Maven 3.9+**
    - **Git**

    Clone the repository and navigate to the project root before running any commands.
  </Step>

  <Step title="Build the project">
    Compile the source and package it into a JAR:

    ```bash
    mvn package
    ```

    This runs all checks and produces `target/*.jar`. To skip tests during the build:

    ```bash
    mvn package -DskipTests
    ```
  </Step>

  <Step title="Run the application locally">
    Start the Spring Boot server on port **8080**:

    ```bash
    mvn spring-boot:run
    ```

    The API is now available at `http://localhost:8080`.

    <Tip>
      If you use VS Code, open the project and press `F5` to start the app with the debugger attached.
    </Tip>
  </Step>

  <Step title="Run the test suite">
    Execute all unit and integration tests (including Cucumber BDD scenarios):

    ```bash
    mvn test
    ```
  </Step>

  <Step title="Register a user">
    User registration is the only endpoint that does **not** require authentication. Send a `POST` to `/users` with a JSON body containing `id` (username), `email`, and `password`.

    ```bash
    curl -X POST http://localhost:8080/users \
      -H "Content-Type: application/json" \
      -d '{
        "id": "alice",
        "email": "alice@example.com",
        "password": "supersecret123"
      }'
    ```

    A successful registration returns **HTTP 201 Created**. The `password` field is never returned in the response.

    <Warning>
      Passwords must be between **8 and 256 characters**. Email must be a valid address. Usernames must be unique. Duplicate usernames or emails return **HTTP 409 Conflict**.
    </Warning>
  </Step>

  <Step title="Call an authenticated endpoint">
    After registering, use HTTP Basic credentials to call protected endpoints. The `/identity` endpoint returns the currently authenticated user:

    ```bash
    curl -u alice:supersecret123 http://localhost:8080/identity
    ```

    Example response:

    ```json
    {
      "id": "alice",
      "email": "alice@example.com",
      "_links": {
        "self": {
          "href": "http://localhost:8080/users/alice"
        }
      }
    }
    ```
  </Step>

  <Step title="Explore the API">
    The backend ships with two browser-based tools for exploring endpoints:

    <CardGroup cols={2}>
      <Card title="HAL Explorer" icon="compass" href="http://localhost:8080">
        Browse the hypermedia-driven REST API. Navigate resource links interactively.
      </Card>
      <Card title="Swagger UI" icon="code" href="http://localhost:8080/swagger-ui.html">
        View auto-generated OpenAPI documentation and try requests directly in the browser.
      </Card>
    </CardGroup>
  </Step>
</Steps>
