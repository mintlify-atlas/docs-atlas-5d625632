A `Floater` is a student volunteer who assists teams on the day of the competition. Floaters extend the [Volunteer](/api/volunteers/volunteers) base entity with a unique `studentCode` and a many-to-many relationship with the teams they assist.

<Note>
  - A team may have **at most 2 floaters** assigned.
  - A floater may assist **multiple teams**.
</Note>

## Floater entity

Inherits all [Volunteer base fields](/api/volunteers/volunteers#volunteer-base-entity), plus:

<ResponseField name="studentCode" type="string" required>
  Unique identifier code for the student volunteer (e.g. `"FLL-001"`). Must be unique across all floaters and cannot be blank.
</ResponseField>

<ResponseField name="assistedTeams" type="object[]">
  The set of teams this floater is currently assisting. Managed via the team resource.
</ResponseField>

## List floaters

```
GET /floaters
```

Returns a paginated list of all floaters. Public endpoint.

### Query parameters

<ParamField query="page" type="integer" default="0">
  Zero-based page number.
</ParamField>

<ParamField query="size" type="integer" default="20">
  Number of results per page.
</ParamField>

<ParamField query="sort" type="string">
  Sort field and direction, e.g. `name,asc`.
</ParamField>

```bash List all floaters
curl -X GET http://localhost:8080/floaters
```

## Create a floater

```
POST /floaters
```

Requires authentication.

### Request body

<ParamField body="name" type="string" required>
  Full name. Cannot be blank.
</ParamField>

<ParamField body="emailAddress" type="string" required>
  Valid, unique email address.
</ParamField>

<ParamField body="phoneNumber" type="string" required>
  Contact phone number. Cannot be blank.
</ParamField>

<ParamField body="studentCode" type="string" required>
  Unique student code (e.g. `"FLL-001"`). Must be unique and cannot be blank.
</ParamField>

```bash Create a floater
curl -X POST http://localhost:8080/floaters \
  -H "Content-Type: application/json" \
  -u admin:admin \
  -d '{
    "name": "Laura Mas",
    "emailAddress": "laura@example.com",
    "phoneNumber": "123456789",
    "studentCode": "FLL-001"
  }'
```

Returns `201 Created` with a `Location` header.

## Update a floater

<CodeGroup>

```bash Full update (PUT)
curl -X PUT http://localhost:8080/floaters/10 \
  -H "Content-Type: application/json" \
  -u admin:admin \
  -d '{
    "name": "Laura Mas",
    "emailAddress": "laura@example.com",
    "phoneNumber": "987654321",
    "studentCode": "FLL-001"
  }'
```

```bash Partial update (PATCH)
curl -X PATCH http://localhost:8080/floaters/10 \
  -H "Content-Type: application/json" \
  -u admin:admin \
  -d '{"phoneNumber": "987654321"}'
```

</CodeGroup>

Returns `204 No Content` on success.

## Team assignment

Floaters are assigned to teams through the Team resource. A team may have at most 2 floaters; a floater may assist any number of teams.

```bash Assign floater to a team (via Team resource)
curl -X PATCH http://localhost:8080/teams/LegoStars/floaters \
  -H "Content-Type: text/uri-list" \
  -u admin:admin \
  -d "http://localhost:8080/floaters/10"
```

<Warning>
  Attempting to assign a third floater to a team will return an error: `"A team cannot have more than 2 floaters"`.
</Warning>
