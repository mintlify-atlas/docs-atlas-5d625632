## Base URLs

| Environment | URL |
|---|---|
| Local development | `http://localhost:8080` |
| Production (Fly.io) | `https://spring-template.fly.dev` |

## API style

The API is built with **Spring Data REST** and follows the **HAL+JSON** hypermedia format (HATEOAS). Most collection and resource endpoints are auto-exposed by the repository layer. Custom business endpoints (lifecycle transitions, team registration, volunteers, tables) are plain REST controllers.

<Note>
  Every response from a Spring Data REST endpoint includes a `_links` object with `self` and relation links. Use these links to navigate the API rather than constructing URLs manually.
</Note>

## Content types

| Direction | Value |
|---|---|
| Request body | `application/json` |
| Response body | `application/hal+json` (Spring Data REST) / `application/json` (custom controllers) |

## Authentication

The API uses **HTTP Basic authentication** over a stateless session. Pass a `Authorization: Basic <base64(username:password)>` header with every request that requires authentication.

```bash
curl -u admin:password https://spring-template.fly.dev/editions
```

Endpoints that require authentication return `401 Unauthorized` when credentials are missing or incorrect.

## Pagination

Spring Data REST collection endpoints support pagination via query parameters.

| Parameter | Default | Description |
|---|---|---|
| `page` | `0` | Zero-based page number |
| `size` | `20` | Number of items per page |
| `sort` | — | Field name and direction, e.g. `year,desc` |

```bash
GET /editions?page=0&size=10&sort=year,desc
```

Paginated responses include a `page` metadata object and `_links` for `first`, `prev`, `self`, `next`, and `last`.

## Error response structure

Custom controllers return errors in the following structure:

```json
{
  "error": "EDITION_NOT_FOUND",
  "message": "Edition with id 42 not found",
  "timestamp": "2025-03-16T10:00:00.000Z",
  "path": "/editions/42/state",
  "details": null
}
```

<ResponseField name="error" type="string" required>
  Machine-readable error code (e.g. `EDITION_NOT_FOUND`, `MAX_TEAMS_REACHED`).
</ResponseField>

<ResponseField name="message" type="string" required>
  Human-readable description of what went wrong.
</ResponseField>

<ResponseField name="timestamp" type="string" required>
  ISO 8601 timestamp of when the error occurred.
</ResponseField>

<ResponseField name="path" type="string" required>
  The request path that produced the error.
</ResponseField>

<ResponseField name="details" type="object">
  Optional structured details (e.g. validation field errors). Omitted when null.
</ResponseField>

## Endpoint index

### Editions

| Method | Path | Auth | Description |
|---|---|---|---|
| `GET` | `/editions` | No | List all editions (paginated) |
| `GET` | `/editions/{id}` | No | Get a single edition |
| `POST` | `/editions` | Yes | Create a new edition |
| `PUT` | `/editions/{id}` | Yes | Replace an edition |
| `PATCH` | `/editions/{id}` | Yes | Partially update an edition |
| `DELETE` | `/editions/{id}` | Yes | Delete an edition |
| `PATCH` | `/editions/{editionId}/state` | Yes | Transition edition lifecycle state |
| `POST` | `/editions/{editionId}/teams/{teamId}` | Yes | Register a team to an edition |
| `GET` | `/editions/{editionId}/volunteers` | Yes | List volunteers grouped by type |
| `GET` | `/editions/{editionId}/tables` | No | List competition tables with scheduled matches |
